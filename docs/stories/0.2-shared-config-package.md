# Story 0.2: Shared Configuration Package (@plpg/config)

## Status
Draft

## Parent Epic
[Epic 0: Technical Foundation](../epic-0-technical-foundation.md)

## Story
**As a** developer,
**I want** centralized configuration for ESLint, TypeScript, and Tailwind,
**So that** all packages use identical standards

## Acceptance Criteria
1. `@plpg/config` package created with exportable configs
2. ESLint config with TypeScript, React, and Prettier rules
3. Base `tsconfig.json` for all packages to extend
4. Tailwind preset with PLPG design tokens

## Tasks / Subtasks
- [ ] Create `packages/config` package (AC: 1)
  - [ ] Initialize package.json for @plpg/config
  - [ ] Set up exports field for config files
- [ ] Define ESLint configurations (AC: 2)
  - [ ] Create `eslint/base.js` with base rules
  - [ ] Create `eslint/react.js` extending base for React apps
  - [ ] Create `eslint/node.js` extending base for Node.js apps
  - [ ] Include TypeScript, Prettier integration
- [ ] Define TypeScript base configs (AC: 3)
  - [ ] Create `typescript/base.json` with strict settings
  - [ ] Create `typescript/react.json` for React apps
  - [ ] Create `typescript/node.json` for Node.js apps
- [ ] Define Tailwind preset (AC: 4)
  - [ ] Create `tailwind/preset.js` with design tokens
  - [ ] Define color tokens: Primary (#1E3A8A), Accent (#10B981), Warning (#F59E0B)
  - [ ] Configure 8px base grid spacing

## Dev Notes

### Package Structure
```
packages/config/
├── eslint/
│   ├── base.js           # Base rules for all packages
│   ├── react.js          # React-specific rules (extends base)
│   └── node.js           # Node.js-specific rules (extends base)
├── typescript/
│   ├── base.json         # Strict TypeScript settings
│   ├── react.json        # React app settings
│   └── node.json         # Node.js app settings
├── tailwind/
│   └── preset.js         # Design tokens, colors, spacing
└── package.json
```

### ESLint Base Config
```javascript
module.exports = {
  extends: [
    'eslint:recommended',
    'plugin:@typescript-eslint/recommended',
    'prettier',
  ],
  plugins: ['@typescript-eslint'],
  rules: {
    '@typescript-eslint/explicit-function-return-type': 'warn',
    '@typescript-eslint/no-unused-vars': ['error', { argsIgnorePattern: '^_' }],
    'no-console': ['warn', { allow: ['warn', 'error'] }],
  },
};
```

### TypeScript Base Config
